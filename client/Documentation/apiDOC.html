GET /channeldata

Gets the channel data for the given channel
This looks up whether the JSON exists in the local JSON files, if not it will run callChannelData to request it from Youtube
If the file exists it will send the file's data as a JSON back as a response
If no channel is provided the 422 error code is provided
Resource URL:
https://trailerscentral.herokuapp.com/channeldata
Parameters:

channel required The channel the user wishes to get trailers for

GET /prefs

Used to get the preferences of a specific user
User is defined by either the req.query.token or req.query.email
I use JSONWebToken to verify the token here -> which hashes the users id into the token
In this instance it unhashes the id out of the token
If the email user is used a new token is provided
Resource URL:
https://trailerscentral.herokuapp.com/prefs
Parameters:
Name Required Description
email Partial requirement (either parameter but not both) The email for the account to get the preferences for
x-access-token (embedded in the header of the request) Partial requirement (either parameter but not both) The token through which the getPrefs method can be made, which belongs to the account linked to the requested preferences
Source:
  • app.js, line 207
Example Request:
fetch("https://trailerscentral.herokuapp.com/prefs",{method:"GET",headers:{"x-access-token":REQUEST_TOKEN}}) OR https://trailerscentral.herokuapp.com/prefs?email=mock%40account.co.uk
Example Response:
											
	{"prefs":["Marvel","Disney","FOX","DC"],"name":"NAME","token":REQUEST_TOKEN}
Type
Object
